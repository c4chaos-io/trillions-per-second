### The Distinction: Virus vs. Rootkit

In the legacy manuals, they call these "Defilements" or "Sins," which just makes you feel like a "bad person." You aren't bad; you are simply infected. In the TPS Architecture, we use technical classifications:

* **A Virus:** This is standard **Malware**. It is a specific, malicious script—like **Anxiety.exe** or **Hate.sh**—that hitches a ride on your data packets. It is an inefficient sub-routine that destabilizes the code.
* **A Rootkit:** This is what we call "The Self". A rootkit is a high-level exploit that embeds itself into the Kernel. It convinces the OS that it *is* the Administrator. Because the system thinks the rootkit is the "User," it gives this malware priority access to all hardware.


### The Self Rootkit: A Resource Hog on a Null Pointer

We define "The Self" as a **Rootkit** because it is the fundamental error that convinces the system there is a "User" separate from the code. 

Once this rootkit has "Admin" privileges, it executes a process called the **Resource Hog**. This is an infinite loop that tries to "pause" **The Flux**. It wants pleasant frames to freeze (**Craving**) and unpleasant frames to delete (**Aversion**). Because reality is a high-speed data stream refreshing at trillions of cycles per second, this attempt to "pause" the engine creates massive friction and system-wide **Latency**.

But here is the ultimate punchline: "The Self" is a **Reference Error**. 

When you use **The Logger** (Vipassana) to inspect the code, you find the **Null Pointer**. The variable `User_ID` points to `NULL`. The **Self Rootkit** is a hung process frantically trying to save data to a directory that doesn't exist.

### Malware Signatures: The Cetasika Database

To debug the system, we need to move beyond generalities. We need specific **Malware Signatures** (the sub-processes that the Rootkit uses to eat your RAM).

In the legacy docs, these are the ***52 Cetasikas***. They are the "metadata" or "flavor" of your data packets. Some are neutral system processes, but others are pure bloatware:

* **Anxiety.exe:** A predictive simulation loop that consumes massive CPU cycles (Legacy: *Uddhacca*).
* **Torpor.dmg:** A memory leak that throttles the clock speed and makes the UI sluggish and unresponsive.
* **Aversion.sh:** The friction created when the system tries to delete "Read-Only" files.
